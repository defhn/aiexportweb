## 3. 我对每个缺失功能的实现选择

### 3.1 询盘数据看板

你的做法里提到用 `/api/dashboard/*`，这不是唯一方式。  
我的选择是：

- V2 先用 `src/features/dashboard/queries.ts` 直接做服务端聚合查询
- 后台首页 Server Component 直接读取这些查询结果
- 如果后面要做可刷新的图表，再补 `/api/dashboard/*`

这样更适合当前架构，因为后台本身就是 Next.js 服务端渲染，不需要先多拆一层 API。

建议新增：

- `src/features/dashboard/queries.ts`
- `src/app/admin/(protected)/page.tsx`

建议查询：

- 今日询盘数
- 本周询盘数
- 本月询盘数
- 未处理询盘数
- 最近 7 天趋势
- 询盘最多产品
- 国家来源分布

### 3.2 产品热度排行

这里有两种实现路线，你上面也写了：

- 路线 A：产品浏览日志
- 路线 B：按询盘统计

我的选择：

- V2 先做路线 B，只统计“带来询盘最多的产品”
- V2.1 再补 `product_views` 表，做浏览热度

原因：

- 现在 `inquiries.productId` 已经存在
- 商业价值更高
- 实现更简单

后续再加：

- `product_views`
  - `id`
  - `product_id`
  - `session_id`
  - `country_code`
  - `referer`
  - `created_at`

### 3.3 产品快速录入系统

当前已经有单个新增/编辑，所以 V2 的重点是批量导入。

你的方案里提到 `CSV / Excel`，这个不是唯一实现。  
我的选择：

- V2 先做 `CSV 导入`
- Excel 留到后续，如果真有客户需求再加 `xlsx`

原因：

- CSV 解析最简单稳定
- 字段映射清晰
- 对外贸客户和助理来说已经够用

建议新增：

- `/admin/products/import`
- `src/features/products/import.ts`
- 下载模板按钮

CSV 模板字段先按你给的来：

- `name_zh`
- `name_en`
- `category`
- `material`
- `process`
- `moq`
- `lead_time`
- `application`

### 3.4 图片一键替换

这个实现方式基本就是你写的那套，我建议直接照搬。

当前已有：

- 图片上传到 R2
- 图库管理
- 产品封面图、图库绑定

V2 要补的是：

- 在产品编辑页每张图旁边加“替换”
- 选新文件后直接调用上传接口
- 上传成功后更新对应图片绑定
- 默认不删旧图

我的选择：

- 不改现有 `media_assets` 设计
- 替换本质是“新建一个 media asset，再把产品关联改到新 asset”
- 这样最安全，也方便后面做未使用素材清理

### 3.10 数据看板国家分布

当前字段里只有原始 `country`，还没做标准化。  
这个功能的关键不是图表，而是先把国家值标准化。

我的选择：

- 新增 `country_code`
- 保留 `country` 原始输入
- 后台统计优先按 `country_code`

这样可以解决：

- `USA`
- `US`
- `United States`

被拆成多个统计项的问题。

### 3.11 文件上传附件到询盘

这个功能不是完全没做，而是已经有基础版。

当前已有：

- 询盘表单附件上传
- 附件上传到 R2
- `attachmentMediaId` 入库

还没做：

- 文件大小限制
- 文件类型白名单
- 上传数量限制
- 后台询盘里展示附件

这里你的方案建议在 `inquiries` 里加 `attachment_url` 和 `attachment_name`。  
这也不是唯一方式。

我的选择：

- 继续保留 `attachmentMediaId`
- 通过关联 `media_assets` 读取 `url / fileName`
- 不重复存 `attachment_url / attachment_name`

原因：

- 避免数据重复
- 文件名和 URL 已经在 `media_assets`
- 后续附件替换或清理更容易维护
