# 公共前台统一收口与验收文档

## 1. 目标

本次工作的目标不是重做系统，而是先把当前站点的前台统一成一套可复用的标准骨架，方便后续扩展 12 个行业子域名站点。

核心原则：

- 统一前台 9 个核心页面的结构与视觉语言
- 保留行业差异，但不破坏整体骨架一致性
- 不改数据库主结构
- 不动后台核心流程
- 不让低套餐页面直接 404 断页

## 2. 已统一的页面范围

本次已对以下页面/区域做了统一收口：

### 2.1 公共页面

- 首页 `/`
- 产品总览 `/products`
- 产品分类页 `/products/[categorySlug]`
- 产品详情页 `/products/[categorySlug]/[productSlug]`
- 能力页 `/capabilities`
- 关于我们 `/about`
- 联系我们 `/contact`
- 博客页 `/blog`
- 报价页 `/request-quote`

### 2.2 公共组件

- `src/components/public/product-card.tsx`
- `src/components/public/blog-card.tsx`
- `src/components/public/category-grid.tsx`
- `src/components/public/inquiry-form.tsx`

### 2.3 模板首页

- `src/templates/template-01/home-page.tsx`

## 3. 本次统一收口的结果

### 3.1 视觉统一

已完成的统一点：

- 标题层级收口
- hero 区视觉节奏统一
- 卡片圆角、边框、阴影统一
- CTA 风格统一
- 页面留白更一致
- 产品与博客卡片更克制、更适合 B2B 官网

### 3.2 功能统一

已完成的统一点：

- `/blog` 不再因套餐限制直接 404
- `/request-quote` 不再因套餐限制直接 404
- 报价接口支持低套餐降级展示
- 表单提交后可提示降级路径

### 3.3 风险控制

本次改动遵守了以下边界：

- 未修改数据库 schema
- 未修改后台主业务流程
- 未改动核心询盘/产品/博客数据结构
- 未引入新的 lint 错误

## 4. 页面级验收结论

### 4.1 首页

首页模板已经从“更强视觉冲击”收敛到“更稳的高端 B2B 首页语言”，更适合作为多行业模板底座。

### 4.2 产品页体系

- `/products` 更像标准目录页
- 分类页更像标准中层流量页
- 产品详情页的面包屑、图片、参数区、下载区与询盘区已统一

### 4.3 信任页与内容页

- `/capabilities`
- `/about`
- `/contact`
- `/blog`
- `/request-quote`

以上页面已向同一套视觉系统靠拢，整体风格更统一。

## 5. 当前验收状态

### 5.1 已通过

- 页面骨架统一
- 公共组件风格统一
- 软门禁策略统一
- lint 检查无新增错误

### 5.2 当前可作为验收标准的结论

- 前台已具备统一骨架
- 后续可以进入 12 个行业模板的行业化配置阶段
- 可以开始按行业逐个落地，而不是继续大范围改前台骨架

## 6. 后续建议

下一阶段不建议继续大幅修改前台公共骨架，而应转入以下工作：

1. 确定 12 个行业的最终名单
2. 为每个行业制定主题色、图片风格、文案方向
3. 复制当前统一好的页面骨架到行业模板
4. 按行业顺序逐个落地（建议先医疗、工业、家居）

## 7. 结论

本次工作已经完成了公共前台的第一轮统一收口。当前系统具备了“一个底座，多行业复用”的基础，可以进入行业模板阶段，而不需要继续重做公共页面结构。
