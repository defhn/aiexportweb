# 外贸获客网站系统设计文档

## 1. 项目定位

本项目是一套面向中国制造业企业的外贸获客网站系统。

它不是普通企业展示站，也不是纯模板站，更不是让客户自己从零搭字段的通用 CMS。它的目标是帮助中国工厂和外贸公司快速搭建一个老板也能自己维护的英文外贸网站，用于展示产品、持续做 SEO、被主流大模型和搜索引擎抓取引用，并稳定沉淀询盘。

一句话定位：

为中国制造业企业提供可管理产品、可沉淀询盘、可持续做 SEO、可被 AI 搜索抓取引用的英文外贸获客网站系统。

## 2. 目标用户

核心用户：

- 中国制造业工厂老板
- 外贸公司负责人
- 需要展示产品、持续更新内容、接收海外询盘的企业

第一阶段非核心用户：

- 同行建站公司
- 代理商
- 复杂 SaaS 客户

## 3. 核心业务目标

系统围绕以下业务链路设计：

Google 和 AI 搜索流量 -> 建立信任 -> 引导询盘 -> 沉淀线索

第一版必须满足 4 个目标：

- 快速生成专业英文外贸站
- 让中国老板通过中文后台自行维护
- 稳定接收并保存询盘
- 能快速复制成多个行业版本交付

## 4. 产品形态

系统采用单站交付模式。

每个客户一套独立部署的网站系统，不做多租户。交付方式是基于母模板复制，并初始化对应行业演示数据包，再替换为客户真实资料。

## 5. 技术架构

- 前端与后台壳子：Next.js + Tailwind CSS
- 部署平台：Vercel
- 业务数据库：Neon
- 图片与文件存储：Cloudflare R2
- 邮件通知：Brevo
- 反垃圾验证：Cloudflare Turnstile

架构原则：

- 所有业务数据统一进入自研后台
- 不使用 Sanity，不使用外部 CMS 后台
- 图片和文件统一托管在 R2
- 邮件只负责通知，询盘数据以 Neon 入库为准

## 6. 语言策略

### 6.1 前台

- 公开网站仅显示英文
- SEO 仅围绕英文页面建设
- 公开路由不做中英双语切换

### 6.2 后台

- 后台界面全部使用中文
- 后台录入内容同时保存中文和英文
- 中文内容用于老板和员工理解、编辑、校对
- 英文内容用于前台展示和 SEO

## 7. 信息架构

### 7.1 前台路由

- `/`
- `/about`
- `/contact`
- `/products`
- `/products/[category-slug]`
- `/products/[category-slug]/[product-slug]`
- `/blog`
- `/blog/[slug]`
- `/privacy-policy`
- `/terms`

### 7.2 URL 策略

产品 URL 采用 `/products/[category-slug]/[product-slug]`。

原因：

- 结构简洁
- 分类语义清晰
- 对 SEO 友好
- 后期扩展稳定

## 8. 前台页面结构

### 8.1 首页

首页是转化页，不是展示页。默认模块如下：

- Hero
- Core Advantages
- Featured Categories
- Featured Products
- Industries We Serve
- Factory Strength
- Certificates
- Case or Clients
- Latest Articles
- Final CTA

首页模块均支持：

- 开启和关闭
- 排序
- 文案编辑
- 图片上传
- 按钮配置
- 推荐产品和推荐分类选择

### 8.2 产品总览页

产品总览页承担广义产品流量入口作用，包含：

- 分类导航
- 推荐分类
- 热门产品
- 产品总览列表
- SEO 文本区

### 8.3 分类页

分类页承担中层商业关键词流量，包含：

- 分类标题和简介
- 产品列表
- 分类级 CTA
- FAQ 或补充说明

### 8.4 产品详情页

产品详情页是整站最核心页面，默认结构如下：

- 产品主图和图库
- 英文产品名称
- 简短卖点
- 关键参数
- Product Overview
- Applications
- Technical Specifications
- Download Area
- Inquiry Card
- Related Products
- FAQ

关键要求：

- 参数表必须以结构化 HTML 展示
- 关键信息不能只放在图片里
- 询盘表单自动带入当前产品名称

### 8.5 About 页面

About 页面是信任页，默认模块包含：

- 公司简介
- 发展历程
- 工厂与设备
- 生产能力
- 核心优势
- 证书展示
- 主要市场
- CTA

### 8.6 Contact 页面

Contact 页面是强转化页，默认模块包含：

- 页面标题与介绍
- 联系方式卡片
- 联系表单
- WhatsApp 快捷联系
- 地址与地图
- 工作时间
- 附件上传说明

### 8.7 Blog

Blog 是 SEO 入口，不是纯资讯模块。

博客列表页包含：

- 分类
- 标签
- 搜索
- 推荐文章
- SEO 引导文本

博客详情页包含：

- 英文标题
- 摘要
- 正文
- 结论或 FAQ
- 关联产品
- CTA

## 9. 全站转化设计

固定转化点：

- 顶部导航按钮
- 页脚 CTA
- 产品卡片按钮
- 产品详情页右侧询盘卡
- WhatsApp 快捷按钮
- Blog 底部 CTA
- Contact 页完整表单

核心转化路径：

- 首页 -> 分类页 -> 产品详情页 -> 询盘
- 首页 -> 推荐产品 -> 产品详情页 -> 询盘
- Blog -> 关联产品 -> 产品详情页 -> 询盘
- 任意页面 -> Contact 或 WhatsApp -> 直接联系

## 10. 后台设计原则

后台必须以中国制造业老板可用为前提。

设计原则：

- 全中文界面
- 表单录入优先
- 少配置，多默认
- 不暴露技术概念
- 不使用页面搭建器
- 页面采用固定模块编辑

## 12. 固定页面编辑模式

固定页面采用表单型模块编辑，不做拖拽式积木搭建。

每个模块支持：

- 开关
- 排序
- 标题和副标题
- 图片
- 按钮
- 关联产品或分类

这样可以保证：

- 老板易用
- 页面结构稳定
- 模板复制成本低
- 不会因误操作破坏页面质量

## 13. 产品系统设计

产品系统采用“系统内置基础字段 + 每产品自定义字段补充”的结构。

### 13.1 基础字段原则

- 系统默认提供完整字段
- 客户无需从零设计字段
- 每个字段支持前台显示或隐藏
- 中文字段用于后台理解
- 英文字段用于前台展示

### 13.2 产品基础信息

- 产品名称中文
- Product Name 英文
- 产品简介中文
- Short Description 英文
- 产品详情中文
- Product Details 英文
- 产品分类
- 产品封面图
- 产品图库
- slug
- 排序值
- 是否发布
- 是否推荐
- SEO 标题
- SEO 描述
- 是否显示 Inquiry 按钮
- 是否显示 WhatsApp 按钮
- 是否显示 PDF 下载按钮

### 13.3 默认参数字段

初始化内置以下默认参数：

- 型号 Model
- 材料 Material

- 尺寸 Size
- 公差 Tolerance

- 颜色 Color

- 包装 Packaging
- 产地 Place of Origin
- 供货能力 Supply Ability
- 认证 Certification

每个参数值都包含：

- 中文值

### 13.4 自定义参数

自定义字段只作为补充，用于特殊行业参数，例如：

每个自定义字段包含：

- 中文名称
- 英文名称

- 排序值

## 18. SEO 与 AI 抓取策略

本项目不只要求对搜索引擎友好，还要求对主流大模型和 AI 搜索友好。

### 18.1 基础要求

- 公开内容使用 SSR 输出
- 关键信息以 HTML 文本呈现
- 图片提供英文 alt
- 页面之间有清晰内链
- 自动生成 sitemap
- 动态输出 robots.txt
- 输出结构化数据

### 18.2 结构化数据

优先输出以下 Schema.org 类型：

- Organization
- Product
- Article
- BreadcrumbList
- FAQPage

### 18.3 AI 抓取后台控制

后台提供“SEO 与 AI 抓取设置”，允许客户分别控制：

- Google 和 Bing 抓取
- OAI-SearchBot
- Claude-SearchBot
- PerplexityBot
- GPTBot
- ClaudeBot

默认策略：

- 搜索型抓取默认开启
- 训练型抓取默认关闭

### 18.4 AI 友好内容原则

- 产品页提供清晰标题、摘要、参数表、应用和 FAQ
- 博客页提供摘要、小标题和结论性内容
- 重要信息不依赖图片
- 页面文本利于抽取和总结

## 19. 后台 AI 设置说明

后台必须用中文解释“搜索引用”和“训练抓取”的差异。

产品建议默认值：

- 允许 Google/Bing 抓取：开启
- 允许 ChatGPT 搜索引用：开启
- 允许 Claude 搜索引用：开启
- 允许 Perplexity 抓取：开启
- 允许 OpenAI 训练抓取：关闭
- 允许 Anthropic 训练抓取：关闭

## 20. 数据模型

### 20.1 核心表

- `admin_users`
- `site_settings`
- `seo_ai_settings`
- `pages`
- `page_modules`
- `product_categories`
- `products`
- `product_default_field_defs`
- `product_default_field_values`
- `product_custom_fields`
- `media_assets`
- `product_media_relations`
- `blog_posts`
- `blog_categories`
- `blog_tags`
- `blog_post_categories`
- `blog_post_tags`
- `blog_post_related_products`
- `inquiries`

### 20.2 页面表

`pages` 存固定页面主信息。

`page_modules` 存页面模块信息，包含：

- 页面标识
- 模块标识
- 模块名称
- 启用状态
- 排序
- 模块内容 JSON

### 20.3 产品表

`products` 存产品主信息。

`product_default_field_defs` 存系统预置字段定义。

`product_default_field_values` 存每个产品的默认参数值。

`product_custom_fields` 存每个产品的自定义参数值。

### 20.4 媒体表

`media_assets` 统一存储图片与文件元信息，通过 `asset_type` 区分。

`product_media_relations` 管理产品和媒体之间的关联、用途和排序。

## 29. 总结

本项目要交付的不是普通网站，也不是通用模板。

它是一套面向中国制造业老板的、开箱即用、统一中文后台管理、英文前台展示、可持续做 SEO、可沉淀询盘、可被 AI 搜索抓取引用、可快速复制交付的外贸获客网站系统。

最重要的产品原则是：

基础字段必须足够完整，客户只需要填空；自定义字段只负责补充行业特殊参数。
