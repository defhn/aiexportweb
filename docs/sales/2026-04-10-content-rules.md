# 抖音、小红书与 PDF 导出统一规则

适用范围：围绕中国制造型出口企业英文官网获客系统的所有内容生产，包括抖音脚本、小红书笔记、标题池扩写，以及 PDF 导出。

## 一、总原则

1. 默认使用中文表达，尽量不要夹带英文。
2. 只有像 `FAQ` 这类用户已经接受的短缩写，可以保留。
3. 其他英文术语尽量改成中文，例如：
   - `Inquiry` -> `询盘`
   - `Quote` -> `报价`
   - `CTA` -> `转化按钮`
   - `Hero` -> `首屏`
   - `About` -> `公司介绍页`
   - `brochure` -> `宣传册`
   - `Google` -> `谷歌` 或 `搜索引擎`
4. 内容必须围绕已实现功能和真实客户痛点，不写空想功能。
5. 语气要像真正在行业里做过事的人，不要像广告文案。
6. 全部内容统一按“40 岁左右的外贸老手口吻”来写，强调经验感、判断感和稳重感。
7. 但不要直接写成“我做过外贸”，也不要刻意强调个人从业经历，重点是写出像资深外贸从业者在分享方法和判断。
8. 每篇内容写完后，必须先做字数校验，确认落在目标区间内，再继续下一篇。
9. 如果字数没有命中目标区间，必须先重写或补充到合格，再进入下一篇，不允许带着不合格版本往下写。

## 二、抖音脚本规则

1. 每个选题单独成段，格式固定：
   - `### 选题 N`
   - 标题
   - `对应已实现功能：...`
   - `建议时长：2 分钟到 4 分钟`
   - `内容：`
2. 正文采用一行一句的口播式排版。
3. 每行之间保留空行，方便后续转 PDF。
4. 内容要像一个有经验的外贸人直接讲给老板听。
5. 每条结尾统一保留：
   - `外贸询盘没着落，胡总局里帮你破。`
6. 不要在正文里混入过多英文词，除非是 `FAQ` 这类已允许缩写。

## 三、小红书笔记规则

1. 每篇按图文笔记来写，不要写成口播稿。
2. 每篇字数建议控制在 `1300-1800 字`。
3. 语气要像一个四十岁左右、做过很多年外贸的真人在分享经验。
4. 必须有有序列表，推荐使用 `1. 2. 3.` 结构。
5. 内容结构建议是：
   - 先指出一个真实问题
   - 再解释为什么会这样
   - 再给判断标准或方法
   - 最后给一个轻咨询动作
6. 每篇最后补 `5 个标签`，标签尽量使用中文。
7. 小红书笔记里尽量不要出现英文词，除了 `FAQ` 这类用户已熟悉缩写。
8. 尽量保持“干货 + 避坑 + 可收藏”的调性。
9. 小红书表达要更像平台内常见的图文笔记风格，可以适当使用更丰富的视觉符号，但不要堆太满。
10. 符号优先使用这些更贴近小红书习惯的写法：
   - `📌` 用于重点提醒
   - `⚠️` 用于避坑提醒
   - `🔍` 用于自查和拆解
   - `🧩` 用于结构拆分和方法论
   - `💡` 用于经验总结和思路提示
   - `❗️` 用于强提醒
   - `👉` 用于引导下一步动作
   - `🌱` 用于成长型、长期型表达
   - `✨` 用于轻一点的亮点提示
   - `✅ / ❌` 仍然可以保留，但不要成为唯一符号
11. 符号的作用是增强阅读节奏，不是装饰。
12. 正文里可以适当出现“建议收藏”“先看这个”“很多老板会卡在这里”这类小红书常见引导语，但整体语气仍要克制、真实、像经验分享。

## 七、PDF 导出规则

1. 不是把 Markdown 原样打印，而是先解析成排版页。
2. `### 选题 N` 不显示在 PDF 正文里。
3. `建议时长` 默认不显示。
4. `对应已实现功能` 要显示。
5. 每篇单独一页。
6. 每页只保留标题、功能说明和正文。
7. 标题居中加粗。
8. 行距和页边距要适合阅读，不要过松。
9. Markdown 符号如 `###`、`---` 不要直接出现在 PDF 中。
10. 生成出来的 PDF 要适合直接发给客户或转交打印。

## 八、统一替换口径

如果后续再扩内容，优先使用下面的中文口径：

1. `Inquiry` -> `询盘`
2. `Quote` -> `报价`
3. `CTA` -> `转化按钮`
4. `Hero` -> `首屏`
5. `About` -> `公司介绍页`
6. `brochure` -> `宣传册`
7. `Google` -> `谷歌` 或 `搜索引擎`

## 九、检查清单

1. 是否有不必要的英文词。
2. 是否有明确的功能对应行。
3. 是否有真实客户痛点。
4. 是否有有序列表或清晰分段。
5. 是否满足当前平台的语气要求。
