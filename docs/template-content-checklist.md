# 模板内容核对清单

这份清单用于每次给客户看模板前快速检查，避免出现跨行业内容或产品细节过早锁死的问题。

---

## 使用方式

给一个模板做展示前，按下面顺序逐项核对：

- 先看行业对不对
- 再看有没有混入别的行业
- 再看产品是否留有后续调整空间
- 最后看客户看到后会不会觉得这是给他们行业准备的

只要某一项不通过，就先不要给客户看。

---

## 1. 行业检查

### 必须确认

- [ ] 首页首屏明确显示当前行业
- [ ] 产品/服务名称属于当前行业
- [ ] 页面语气符合当前行业气质
- [ ] 当前行业的应用场景是合理的
- [ ] 当前行业的信任点是合理的

### 不允许出现

- [ ] 其他行业名称
- [ ] 其他行业产品
- [ ] 其他行业案例
- [ ] 其他行业术语
- [ ] 其他行业风格图片

---

## 2. 首页检查

### 必须确认

- [ ] 首页标题是当前行业相关内容
- [ ] 首页副标题没有写死过细的产品型号
- [ ] 首屏图或背景图符合当前行业
- [ ] 首页模块顺序看起来合理
- [ ] 首页 CTA 符合当前行业采购逻辑

### 不允许出现

- [ ] 医疗类词汇出现在非医疗行业首页
- [ ] 照明类词汇出现在非照明行业首页
- [ ] 家居类词汇出现在非家居行业首页
- [ ] 与行业无关的案例图

---

## 3. 产品页检查

### 必须确认

- [ ] 产品分类属于当前行业
- [ ] 产品名称是行业内部通用说法
- [ ] 产品描述没有提前锁死到单一型号
- [ ] 参数、材质、应用场景都符合当前行业

### 不允许出现

- [ ] 别的行业产品混入产品页
- [ ] 不相关的参数表
- [ ] 不相关的产品图片
- [ ] 明显不属于当前行业的命名方式

---

## 4. 分类页检查

### 必须确认

- [ ] 分类维度属于当前行业内部逻辑
- [ ] 分类名称不会让客户出戏
- [ ] 分类之间的层级关系合理

### 不允许出现

- [ ] 分类逻辑直接跳到其他行业
- [ ] 分类名称带出别的行业语义

---

## 5. FAQ 检查

### 必须确认

- [ ] FAQ 问的是当前行业真实会问的问题
- [ ] FAQ 能支持客户采购决策
- [ ] FAQ 不会引导到别的行业场景

### 不允许出现

- [ ] 医疗认证类问题出现在非医疗行业
- [ ] 照明参数类问题出现在非照明行业
- [ ] 其他行业常见问题混入当前 FAQ

---

## 6. 博客检查

### 必须确认

- [ ] 博客主题围绕当前行业展开
- [ ] 博客标题与当前行业相关
- [ ] 博客内容方向能服务 SEO 或客户教育

### 不允许出现

- [ ] 其他行业知识文章
- [ ] 其他行业评测或科普
- [ ] 与当前行业无关的内容方向

---

## 7. 图片和视觉检查

### 必须确认

- [ ] 图片属于当前行业场景
- [ ] 视觉风格符合当前行业气质
- [ ] 配色没有明显串行业感

### 不允许出现

- [ ] 明显不相关的图库图
- [ ] 与当前行业冲突的设备/场景图
- [ ] 医疗、家居、照明等跨行业视觉素材混入

---

## 8. 通用层检查

### 可以先通用的内容

- [ ] 页面布局
- [ ] 导航结构
- [ ] 按钮位置
- [ ] 基础表单逻辑
- [ ] 页脚结构

### 不要通用死的内容

- [ ] 首屏文案
- [ ] 产品分类
- [ ] FAQ
- [ ] 案例
- [ ] 博客主题
- [ ] CTA 目标

---

## 9. 产品细化空间检查

### 必须确认

- [ ] 当前模板没有把产品写死到不可修改
- [ ] 客户确认 A 产品或 B 产品后还能继续改
- [ ] 当前版本保留了后续定制空间

---

## 10. 最终放行标准

如果下面 4 条都成立，就可以给客户看：

- [ ] 行业正确
- [ ] 没有跨行业污染
- [ ] 产品细节留有后续调整空间
- [ ] 客户看了会觉得这是给他们这个行业准备的

---

## 一句话结论

只要行业对、内容不串、产品能后改，这个模板就可以先拿去和客户沟通。
